UI/UX Iteration U2.2: Branch switches refresh all branch-aware routes correctly.
- Root cause: Products page (#/products) had an infinite API loop on branch switch.
  - `renderCatProducts()` detected branch change (`st.companyId !== currentCompanyId`)
    and reset `st.loading = false`, but `st.companyId` was only updated after API response.
  - `loadCatProductsData()` called `renderCatProducts()` in loading state, which re-detected
    the branch mismatch (stale `st.companyId`), reset `st.loading = false`, and re-triggered
    `loadCatProductsData()` → infinite loop (1928+ API calls per branch switch).
- Fix: Set `st.companyId = currentCompanyId` immediately in `renderCatProducts()` when
  branch change is detected, before any load is triggered (app.js line 9633).
